Gridea + Github Pages 搭建个人博客 - 获取Github令牌号教程

要使用Gridea管理你的GitHub Pages博客，你需要获取GitHub的个人访问令牌（Personal Access Token）。以下是详细的获取步骤：

1. 登录GitHub

首先确保你已经登录到GitHub账号。

2. 创建新的访问令牌

  1. 点击右上角你的头像
  2. 选择 "Settings"（设置）
  3. 在左侧边栏底部找到 "Developer settings"（开发者设置）
  4. 点击 "Personal access tokens"（个人访问令牌）
  5. 选择 "Tokens (classic)"（经典令牌）
  6. 点击 "Generate new token"（生成新令牌）
  7. 选择 "Generate new token (classic)"（生成经典令牌）

3. 配置令牌信息

  1. 在 "Note"（备注）处填写一个便于识别的名称，比如 "Gridea"
  2. 设置令牌的有效期（建议选择 "No expiration"，即永不过期）
  3. 权限选择：为了确保Gridea能够正常管理你的GitHub Pages，建议将所有权限都勾选上

4. 保存令牌

  1. 点击页面底部的 "Generate token"（生成令牌）按钮
  2. 重要提醒：请立即复制并保存生成的令牌，因为关闭页面后你将无法再次查看它

5. 在Gridea中使用令牌

  1. 打开Gridea软件
  2. 进入设置页面
  3. 在「远程」设置中：
    • 填写你的GitHub用户名
    • 将刚才保存的令牌粘贴到「Token」输入框中
    • 填写你的GitHub Pages仓库名（通常是 username.github.io）
    • 分支选择 main 或 master

6. 测试连接

配置完成后，点击「检测远程连接」按钮测试连接是否正常。如果显示连接成功，就说明令牌配置正确，Gridea已经成功连接到你的GitHub Pages

注意事项

  1. 请妥善保管你的访问令牌，不要分享给他人
  2. 如果令牌泄露，请立即到GitHub中删除该令牌并创建新的
  3. 如果你在使用过程中遇到同步问题，可以尝试重新生成令牌

完成以上步骤后，你就可以使用Gridea来管理和发布你的博客文章了
